Website information

Accessibility

The website has been designed with WCAG 2.2 Level AA principles in mind, including mobile use, keyboard access and assistive technologies.

01
Standard applied

Readability, predictability and mouse-free operation

Page structure, contrast, heading hierarchy and interactive elements have been prepared with WCAG 2.2 AA in mind.

A renewed automated and manual test is recommended after deployment, as hosting configuration or subsequent content changes may affect the result.
02
Implemented features

Key accessibility measures

Measures cover both the website code and the way content is presented.

Keyboard navigationVisible focus, logical focus order and menu operation with Tab and Escape.
Large touch targetsButtons and menu items are at least 48 × 48 CSS pixels.
Semantic structureOne H1 heading, logical H2 and H3 levels, and header, nav, main and footer landmarks.
Contrast and scalingText meets contrast requirements and the layout supports zoom and narrow screens.
Reduced motionThe system “reduce motion” setting disables smooth scrolling and transitions.
Descriptions and labelsImages have alternative text and controls and navigation areas have clear labels.
03
Reporting issues

Help us improve accessibility
