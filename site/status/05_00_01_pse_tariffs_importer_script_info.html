Importer PSE RCE i PDGSZ — Pobiera publiczne dane PSE RCE oraz PDGSZ, scala je do obowiązkowego pliku 05_pse_tariff.csv i publikuje czyste wartości liczbowe do centralnej tabeli automatyki.

Co robi ten skrypt

Skrypt pobiera dwa publiczne strumienie danych PSE: RCE, czyli rynkową cenę energii, oraz PDGSZ, czyli sygnał/strefę dystrybucyjno-systemową.

Głównym i obowiązkowym wynikiem jest scalony plik 05_pse_tariff.csv. Plik jest budowany w slotach date + start + end, tak aby mógł być używany przez dalsze moduły automatyki energii.

Dane wejściowe

  • API PSE RCE: publiczne dane ceny energii w PLN/MWh, przeliczane do PLN/kWh.
  • API PSE PDGSZ: publiczne dane statusu/strefy systemowej.
  • /work/01_setup/05_01_pse_PDGSZ.csv: lokalne mapowanie statusów PDGSZ na kody PDGSZ_* oraz stawki dystrybucyjne.

Wyniki

  • Must-have: /work/05_tariffs/05_pse_tariff.csv.
  • /work/80_automation/80_automation_data.csv: publikacja kolumn numerycznych pse_RCE i pse_PDGSZ.
  • /work/80_automation/80_automation_data_now.csv: bieżący widok aktualnego slotu, wyprowadzony z centralnej tabeli.
  • Status HTML i alert TXT zgodnie ze standardem NIA status/script_info/alert.

Logika działania

RCE jest pobierane dla dzisiaj i jutra, a okresy godzinowe są rozbijane na sloty 15-minutowe. PDGSZ jest pobierany dla dzisiaj i jutra oraz wzbogacany mapowaniem z pliku setupowego.

Następnie skrypt scala oba strumienie po kluczu date + start + end. W trybie append istniejący plik 05_pse_tariff.csv jest zachowywany, a nowe rekordy są upsertowane po kluczu slotu.

Starsze rozdzielone produkty RCE/PDGSZ typu *_today.csv, *_tomorrow.csv i *_horizon.csv są usuwane po poprawnym zapisie scalonego pliku, aby nie zaśmiecały warstwy taryfowej.

Jak interpretować status

  • OK oznacza, że skrypt zaktualizował obowiązkowy plik 05_pse_tariff.csv i wyczyścił aktywne alerty tego komponentu.
  • ERROR oznacza błąd pobrania, konfiguracji, mapowania, zapisu lub statusu. Przy błędzie powstaje alert TXT.
  • Brak danych na jutro nie musi być błędem, jeżeli PSE nie opublikowało jeszcze horyzontu jutra. Błędem jest brak wymaganych wierszy RCE dla dzisiaj.

Źródła i ograniczenia

Źródłem RCE i PDGSZ są publiczne endpointy PSE skonfigurowane w sekcjach [rce] i [pdgsz]. Dostępność danych zależy od publikacji po stronie PSE.

Skrypt nie podejmuje decyzji sterujących. Przygotowuje warstwę danych taryfowych dla modułów decyzyjnych i raportowych.
